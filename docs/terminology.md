# Terminology

This document defines normative keywords used in ANR documentation.

## MUST

`MUST` indicates a mandatory requirement.
A repository that violates a MUST requirement is not compliant at the stated level.

## SHOULD

`SHOULD` indicates a strong recommendation.
There may be valid reasons to deviate, but deviations should be explicit and justified.

## MAY

`MAY` indicates an optional capability.
It can improve portability or usability but is not required for compliance.

These terms are used to distinguish required and optional ANR components.
