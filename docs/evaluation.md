# Evaluation Design

This document outlines how ANR repositories can be evaluated experimentally.

## Experimental setup

Compare baseline repositories (without ANR structure) against ANR repositories with equivalent codebases and tasks.

## Suggested metrics

- task completion rate
- defect rate
- review latency
- prompt count

## Measurement guidance

1. Use the same task set for baseline and ANR conditions.
2. Keep model and runtime settings fixed where possible.
3. Record outcomes across multiple repositories and domains.
4. Analyze both success quality and operational cost.

The objective is to test whether structured repository context improves agent reliability and collaboration efficiency.
